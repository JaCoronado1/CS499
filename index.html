Enhancements Narrative

üéØ Software Design and Engineering Enhancement

View the Design Enhancement Repository

To improve the structure and maintainability of my food inventory application, I applied software design and engineering principles to refactor and organize the codebase. Initially, much of the business logic, UI interaction, and data handling were all contained within a single activity, leading to tightly coupled and hard-to-maintain code. I redesigned the project using the Model-View-Controller (MVC) architecture, separating data handling from the user interface and core logic. I also created reusable helper classes and modular components to improve readability and scalability, making the application easier to update and maintain in the future.

‚öôÔ∏è Algorithms and Data Structures Enhancement

View the Algorithms and Data Structures Enhancement Repository

The original application used simple, linear search techniques to filter and sort inventory items. As the dataset grew, performance issues began to emerge. To address this, I optimized the expiration check algorithm by pre-sorting inventory data by date and applying a sliding window filter to focus only on soon-to-expire items. I also introduced a HashMap to categorize and retrieve items more efficiently based on their properties, allowing for constant-time access. These enhancements reduced the application's load time and significantly improved user experience when managing large inventories.

üóÑÔ∏è Database Enhancement

View the Database Enhancement Repository

To strengthen the application‚Äôs data layer, I redesigned the SQLite database schema for better normalization and scalability. The original design stored category and unit data directly in the items table, which risked data redundancy and inconsistencies. I refactored the schema by creating separate tables for categories and units, then implemented foreign key relationships to link them to the inventory table. I also added constraints and indexes to enforce data integrity and improve query performance, making the database structure more robust and efficient for future expansion.
